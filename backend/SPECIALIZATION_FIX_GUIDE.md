# Specialization System - Complete Fix Guide

## Problem Identified
- DOC_SPECIALIZATION.ID has NO auto-increment (no sequence/trigger)
- Backend INSERT doesn't specify ID
- Oracle throws error: "ORA-01400: cannot insert NULL into ID"

## Database Structure

### SPECIALIZATION (Master Table)
```
ID              NUMBER NOT NULL (PK)
ADMIN_ID        NUMBER NOT NULL
NAME            VARCHAR2(100) NOT NULL
DESCRIPTION     VARCHAR2(500)
```
- Stores all available specializations (Cardiology, Neurology, etc.)
- Managed by admin
- Current max ID: 10

### DOC_SPECIALIZATION (Junction Table)
```
ID                  NUMBER NOT NULL (PK)
DOCTOR_ID           NUMBER NOT NULL (FK → DOCTOR.ID)
SPECIALIZATION_ID   NUMBER NOT NULL (FK → SPECIALIZATION.ID)
```
- Links doctors to their specialization
- One doctor = one specialization (enforced by DELETE + INSERT)
- Current max ID: 9

## Solution: Run This SQL

**File:** `backend/FIX_SPECIALIZATION_AUTO_ID.sql`

This will:
1. Create SPECIALIZATION_SEQ starting at 11
2. Create DOC_SPECIALIZATION_SEQ starting at 10
3. Create triggers to auto-generate IDs on INSERT
4. Verify triggers are enabled

## Backend Logic (Already Correct)

**File:** `backend/controllers/doctorSpecialization.js`

```javascript
// Update specialization
exports.saveDoctorSpecialization = async (req, res) => {
  // 1. Get doctor ID from email
  // 2. DELETE existing row: DELETE FROM DOC_SPECIALIZATION WHERE DOCTOR_ID = :doctorId
  // 3. INSERT new row: INSERT INTO DOC_SPECIALIZATION (DOCTOR_ID, SPECIALIZATION_ID) VALUES (...)
  //    ↑ ID is NOT specified - will be auto-generated by trigger
  // 4. COMMIT
}
```

**Why DELETE + INSERT?**
- Ensures one doctor has only one specialization
- Simpler than UPDATE (no need to check if row exists)
- Cleaner than MERGE

## Error Handling

If you still get errors after running the SQL:
- Check trigger status: `SELECT trigger_name, status FROM user_triggers WHERE table_name = 'DOC_SPECIALIZATION';`
- If DISABLED, enable: `ALTER TRIGGER TRG_DOC_SPECIALIZATION_ID ENABLE;`
- Check sequence exists: `SELECT sequence_name FROM user_sequences WHERE sequence_name = 'DOC_SPECIALIZATION_SEQ';`
